# 06. Админ-панель и Процессы (Admin Workflow)

Документ описывает функционал закрытой части системы, предназначенной для сотрудников приюта.
**Технический стек:** React (Refine framework) + PWA (Progressive Web App).
**Ключевой принцип:** "Mobile First". 80% действий волонтеры выполняют с телефона.

## 1. Структура и Навигация

### 1.1. Дашборд (Главная)
Сводка оперативной информации при входе.
* **Блок "Внимание":**
    * 🔴 "Не отправлены отчеты опекунам: 3 шт." (Кликабельно -> фильтр списка).
    * 🟠 "Заканчивается корм: 2 позиции".
* **Блок "Мои задачи":**
    * Новые заявки на знакомство (если я дежурный).
* **Быстрые действия (FAB - Floating Action Button):**
    * 📸 Добавить фото (открывает камеру).
    * 📝 Создать новость.

## 2. Управление Животными (Animals CRUD)

### 2.1. Создание карточки (Прием животного)
Процесс максимально упрощен для полевых условий.
1.  **Фото:** Загрузка мульти-селектом или съемка камерой. Авто-ресайз на клиенте перед отправкой.
2.  **Основные данные:** Имя, Пол (переключатель), Вид, Дата рождения (календарь).
3.  **Статус:** По умолчанию ставится `DRAFT` (Черновик) или `QUARANTINE` (Карантин).
4.  **Описание:** Волонтер может надиктовать текст голосом (speech-to-text телефона), потом поправить.

### 2.2. Редактирование и Статусы
* **Смена статуса:** В один клик. (Например, "Ищет дом" -> "На испытательном").
* **Галерея:** Возможность менять порядок фото (drag-and-drop), назначать "Главное фото".

## 3. Работа с Опекунами (CRM)

### 3.1. Отправка отчетов (Monthly Report)
Ключевой процесс удержания доноров.
1.  Волонтер заходит в раздел "Опекунство".
2.  Видит список животных, у которых подходит срок отчета (подсвечены желтым/красным).
3.  Нажимает "Отправить отчет".
4.  **Форма:**
    * Загрузка медиа (Фото/Видео).
    * Текстовое поле: "Как дела у питомца".
    * Кнопка: "Отправить".
5.  **Результат:** Система сама находит Telegram-чат опекуна и шлет контент. В админке ставится галочка "Отчет за Февраль сдан ✅".

## 4. Склад и Нужды (Inventory Lite)
Упрощенное управление списком "Что нужно".
* Таблица товаров.
* Колонка "Статус" (переключатель): `ОК` / `НУЖНО` / `СРОЧНО`.
* Изменение статуса мгновенно обновляет список на сайте.

## 5. Администрирование (Admin Only)

### 5.1. Управление командой
* Список пользователей системы.
* Кнопка "Пригласить волонтера" (Email + Роль).
* Блокировка доступа (при увольнении).

### 5.2. Модерация контента
* Админ видит ленту изменений: "Волонтер Анна изменила описание у Рекса".
* Может откатить изменения.

### 5.3. Финансы (Export)
* Просмотр списка транзакций (Read-only).
* Кнопка "Скачать отчет за месяц (XLSX)" — генерирует сводный документ для бухгалтерии/публикации.

## 6. UX/UI Требования
* **Авторизация:** Долгая сессия (Remember me), чтобы не логиниться каждый раз на морозе.
* **Offline Mode:** (Желательно в v2) Возможность создать черновик карточки без интернета и отправить, когда появится связь.
* **Темная тема:** Автоматически, для экономии батареи телефона, без светлой темы.
