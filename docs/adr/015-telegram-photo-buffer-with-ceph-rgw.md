# ADR-015: Отправка фото в Telegram через Buffer при создании поста (Ceph RGW / WEBPAGE_MEDIA_EMPTY)

- **Status:** Accepted
- **Date:** 2026-04-21
- **Author:** Backend Team
- **Context:**
  При отправке поста в Telegram из `apps/api/src/app/telegram-bot/telegram-bot.service.ts` фото, переданные как URL из S3-совместимого хранилища (Ceph RGW), периодически падают с ошибкой Telegram API: `400 WEBPAGE_MEDIA_EMPTY`.
  Это типичная проблема для Ceph RGW: Telegram не всегда может корректно подтянуть изображение по внешней ссылке (доступность URL, редиректы, заголовки/метаданные, особенности обработки источника на стороне Telegram).
  Для видео такого поведения не наблюдается: видео стабильно отправляются через текущий S3-подход.

- **Decision:**
  Для **фото** при отправке поста в Telegram используем загрузку контента в `Buffer` и отправляем бинарные данные напрямую в Telegram API, вместо передачи URL.
  Для **видео** сохраняем текущий подход: отправка через S3/URL, как реализовано сейчас.

- **Reasons:**
    1. Telegram при отправке фото по URL к Ceph RGW может возвращать `WEBPAGE_MEDIA_EMPTY`, даже когда файл физически существует в бакете.
    2. Передача `Buffer` исключает зависимость от того, как Telegram интерпретирует внешний URL и метаданные ответа RGW.
    3. Для видео проблема не воспроизводится, поэтому нет смысла усложнять и удорожать pipeline без необходимости.
    4. Разделение стратегии по типам медиа минимизирует риск регрессий и сохраняет текущую производительность там, где всё уже работает.

- **Consequences:**
    - **Positive:**
        - Существенно снижается вероятность `400 WEBPAGE_MEDIA_EMPTY` при публикации постов с фото.
        - Поведение отправки фото становится детерминированным и меньше зависит от особенностей Ceph RGW.
        - Видео-пайплайн не меняется, сохраняется текущая стабильная логика.
    - **Negative:**
        - Для фото появляется дополнительная нагрузка на API-процесс (скачивание объекта в память перед отправкой).
        - При больших изображениях выше пиковое потребление RAM, требуется учитывать лимиты и мониторинг.

- **Compliance:**
    - В `apps/api/src/app/telegram-bot/telegram-bot.service.ts` фото для отправки постов должны уходить в Telegram как бинарные данные (`Buffer`), а не как внешний URL.
    - Видео в том же сценарии должны продолжать отправляться через существующий S3/URL-механизм.
    - При рефакторинге сервиса нельзя унифицировать фото и видео в URL-стратегию, пока сохраняется риск `WEBPAGE_MEDIA_EMPTY` для Ceph RGW.
