# ADR-007: Telegram-бот как модуль в api (nestjs-telegraf)

- **Status:** Accepted
- **Date:** 2026-03-07
- **Author:** Backend Team
- **Context:**
  Для опекунов нужен Telegram-бот: привязка по ссылке из письма, команды /my_animals, /help, /unsubscribe, рассылка отчётов из админки. Возник выбор: реализовать бота отдельным приложением (второй процесс/сервис) или модулем внутри существующего приложения api.

- **Decision:**
  Бот реализуется **модулем внутри приложения api** (apps/api), а не отдельным приложением. Стек: **nestjs-telegraf** для интеграции Telegraf с NestJS (декораторы, DI, единый lifecycle).
  Обработчики бота инжектят CommandBus, QueryBus, сервисы и репозитории и вызывают команды/запросы напрямую. Режим работы — long polling (при необходимости позже возможен webhook).

- **Reasons (почему не выносить в отдельное приложение):**
    1. **Одно подключение к БД** — модуль бота использует тот же DatabaseService и пул соединений; нет второго процесса с отдельным DATABASE_URL.
    2. **Без контрактов между приложениями** — не нужны отдельные DTO/схемы и HTTP- или message-контракты для обмена между api и ботом.
    3. **Единый деплой и конфигурация** — один образ/процесс, один набор env, одна точка входа; проще CI/CD и откат.
    4. **Переиспользование CQRS** — бот вызывает те же команды и запросы, что и HTTP-api; нет дублирования бизнес-логики.
    5. **Меньше инфраструктуры** — не нужен отдельный хостинг для бота, внутренняя сеть и секреты для межсервисной аутентификации.
    6. **Консистентность транзакций и кэша** — общие транзакции и инвалидация кэша в одном процессе без распределённых сценариев.

- **Consequences:**
    - **Positive:**
        - Один процесс, один пул БД, единый деплой и конфиг.
        - Прямой вызов команд/запросов из обработчиков бота без HTTP и контрактов.
        - Использование NestJS DI и nestjs-telegraf (декораторы, интеграция с lifecycle).
    - **Negative:**
        - Бот и HTTP-api масштабируются вместе (одни реплики); при очень высокой нагрузке именно на бота независимое масштабирование невозможно без выноса в отдельный сервис (при необходимости внутренний контроллер и guard позволят перейти на HTTP-вызовы).

- **Compliance:**
    - Код Telegram-бота размещается только в модуле `apps/api/src/app/telegram-bot/`.
    - Не создавать отдельное приложение (apps/telegram-bot или аналог) для бота без пересмотра данного ADR.
