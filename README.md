# ПИФ

Веб-платформа для **приюта бездомных животных «ПИФ»** (Донецк).

**Миссия:** Цифровизация процессов приюта для увеличения числа пристройств и обеспечения финансовой прозрачности. Платформа служит мостом между животными и потенциальными хозяевами и донорами.

**Целевая аудитория:** потенциальные хозяева (поиск питомца), доноры (пожертвования, прозрачность расходов), волонтёры (ведение базы и отчётов с телефона).

**Дизайн (Figma):** [ПИФ — Сайт](https://www.figma.com/design/UqKc0Mvsw0Fe6aioX2dUXd/%D0%9F%D0%98%D0%A4-%D0%A1%D0%B0%D0%B9%D1%82?node-id=0-1&t=2fXD9qfKRb5iuQbd-1)

## Стек

- **Монорепозиторий:** [Nx](https://nx.dev), pnpm
- **API:** NestJS 11, Fastify, CQRS, Better Auth
- **Web (публичный сайт) и Admin:** React 19, Vite, TanStack Query, React Router, Tailwind CSS 4, React Hook Form + Zod, TipTap (rich text)
- **БД:** PostgreSQL 17, [Drizzle ORM](https://orm.drizzle.team)
- **Кэш и очереди:** Redis (cache + BullMQ)
- **Файлы:** S3-совместимое хранилище (presigned upload)
- **Почта:** Nodemailer + React Email
- **Бот:** Telegraf (nestjs-telegraf)
- **Контракты и валидация:** Zod, libs/contracts

Архитектура: modular monolith, гибридный repository pattern (write — репозитории, read — прямые запросы в Query Handlers), cache-aside для чтения, инвалидация кэша через доменные события.

## Готовый функционал

### API

- Каталог и карточки животных, медиа, заявки на знакомство.
- Посты по животному: CRUD для волонтёров (куратор/старший/админ), приватные/публичные, до 10 медиа (≤1 видео), HTML-тело санитизируется через `sanitize-html`. Анонимные реакции по 6 эмодзи (`🤪 😍 😂 😱 😭 😎`) с toggle на повторное нажатие; идентификатор посетителя — заголовок `X-Anonymous-Visitor-Id`.
- Пожертвования, платёжные вебхуки, опекунство, кампании сборов и список нужд.
- Чёрный список и блокировки опекунства, отчёты опекунам.
- Финансовая выгрузка в `XLSX` (месячные отчёты + ссылки на скачивание).
- Админский контур пользователей: приглашения в команду, принятие инвайта, управление флагом `telegram_unreachable`.
- Telegram-бот для опекунов: привязка по одноразовому токену, `/my_animals`, `/report`, `/help`, `/unsubscribe` (отмена опекунства). Уведомления о новых постах животного уходят активному опекуну через очередь `POSTS_QUEUE_NAME`.
- Отмена рекуррентных донатов — через PSP и email-ссылки, не через Telegram-бот.

### Web (публичный сайт)

- Каталог и карточка животного, заявки на знакомство, оформление опекунства с авторизацией.
- Дневник животного: таймлайн постов с переключателем по годам, инфинит-скроллом, галереей медиа и анонимными реакциями.
- Список донатов и сборов, кампании, страницы «О нас», FAQ, условия и политика.

### Admin (админ-панель)

- Управление животными, статусами, опеками и встречами.
- Редактор постов (TipTap) с управлением медиа, видимостью, привязкой к животному.
- Финансовый блок: список донатов, кампании, генерация месячных отчётов.
- Управление командой и приглашениями.

## Быстрый старт

### Требования

- Node.js 20+
- Docker и Docker Compose (для БД и Redis)
- Файл окружения для API (см. ниже)

### 1. Установка зависимостей

```bash
pnpm ci
```

### 2. Окружение

Создай `envs/.api.env` и `.env` (для docker compose) скопировав из примера.

### 3. Инфраструктура (БД, Redis)

```bash
docker compose up -d
```

Поднимаются: `database` (5432), `redis` (6379), `queue-redis` (6380). Для разработки можно поднять только их и запускать API локально.

### 4. Миграции

Применить существующие миграции:

```bash
npx nx run database:migrate
```

Создать новую миграцию (после изменения схем в `libs/database`):

```bash
npx nx run database:generate -- --name "название_миграции"
```

### 5. Запуск приложений

API:

```bash
npx nx serve api
```

Публичный сайт (Vite dev server):

```bash
npx nx serve web
```

Админ-панель:

```bash
npx nx serve admin
```

API будет доступен на порту из конфига (например 3000). Health: например `GET /health`.

## Полезные команды Nx

| Действие          | Команда                    |
| ----------------- | -------------------------- |
| Сборка API        | `npx nx build api`         |
| Сборка web        | `npx nx build web`         |
| Сборка admin      | `npx nx build admin`       |
| Тесты API         | `npx nx test api`          |
| Линт всего        | `npx nx run-many -t lint`  |
| Тайпчек web       | `npx nx run web:typecheck` |
| Граф зависимостей | `npx nx graph`             |

Миграции БД создаются только через:

```bash
npx nx run database:generate -- --name "название_миграции"
```

SQL-миграции вручную не пишем.
